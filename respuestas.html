  1. Internet es una red global de computadoras conectadas y www es el conjunto de informacion y servicios accesibles a través de la internet

  2. URL(Uniform Resource Locator) esta conformado por:

    • Protocolo: Indica el método de transferencia de datos (HTTPS, HTTP, FTP, etc.).
    • Subdominio: Agrupa contenido dentro del dominio principal (blog, tienda, etc.).
    • Dominio: Nombre único que identifica al sitio web (wikipedia.org).
    • Dominio de Nivel Superior (TLD): Categoriza el tipo de sitio web (.com, .org, .edu, etc.).
    • Ruta: Directorio o archivo específico dentro del sitio web (carpeta/subcarpeta/archivo.html).
    • Parámetros: Información adicional para la página (filtros, búsquedas, etc.).
    • Fragmento: Ubicación específica dentro de la página (secciones, anclas, etc.).

    protocolo://nombre_de_dominio[:puerto]/ruta?parámetros#fragmento

    • GET: Se utiliza para recuperar un recurso del servidor.

    • HEAD: Solo devuelve la cabecera del recurso, sin el cuerpo.

    • POST: Se utiliza para enviar datos al servidor.

    • PUT: Se utiliza para actualizar un recurso existente en el servidor.

    • PATCH: Se utiliza para aplicar modificaciones parciales a un recurso.

    • DELETE: Se utiliza para eliminar un recurso del servidor.

  4. Se debe utilizar el método POST al enviar un formulario HTML. Este método envia los datos del formulario de manera segura y privada al servidor web, ya que los datos son enviados en el cuerpo de la solicitud HTTP en lugar de ser mostrados directamente en la URL como en el método GET. Esto ayuda a proteger información sensible como contraseñas.
  5. Se utiliza el método GET cuando a través de un navegador web se accede a una página a través de un URL. Este método se utiliza para recuperar un recurso del servidor.
  6. Una respuesta HTTP con código 200 indica que la solicitud fue exitosa. No ocurrió ningún error
  7. Sí, es responsabilidad del desarrollador corregir un error 404 en un sitio web. El error 404 indica que la página solicitada no fue encontrada, lo que puede deberse a una falta de mantenimiento o a un problema de diseño del sitio.
  8. Sí, es responsabilidad del desarrollador corregir un error 500 en un sitio web. El error 500 indica un problema interno del servidor, que puede ser causado por errores en el código o configuración del servidor, y afecta la funcionalidad del sitio para los usuarios.

  10. Existen múltiples diferencias entre HTML 4 y 5 algunas de ellas son:

    Html 4 necesita plugins para mostrar y reproducir multimedia como Flash, html 5 no necesita plugins para reproducir contenido multimedia ya que este de manera nativa lo soporta. En html 5 existe web storage y local storage lo que permite almacenar datos de un cliente, html 4 no tiene almacenamiento local. Html 5 implementa un mayor numero de etiquetas que mejoran la accesibilidad, contenido y el SEO del sitio web. Los formularios de html 4 son más limitados y no tienen validación nativa. Html 5 elimina la mayoría de las etiquetas obsoletas de html 4, su sintaxis es mas legible y tiene una mayor compatibilidad con navegadores modernos.

  11. Una tabla en HTML tiene los siguientes componentes de estructura y estilo: 

                            Componentes de Estructura:
                        <table>: Define la tabla.
                        <tr>: Define una fila dentro de la tabla.
                        <th>: Define una celda de encabezado.
                        <td>: Define una celda de datos.
                        Componentes de Estilo:
                        CSS: Se puede aplicar CSS para estilizar la tabla, incluyendo propiedades como border, background-color, font-family, padding, margin, entre otros.

  12. Los principales controles de un formulario HTML5 son: 

                            1. <input>: Para ingresar texto, números u otros datos.
                        2. <textarea>: Para ingresar texto largo.
                        3. <select>: Para crear menús desplegables.
                        4. <button>: Para enviar el formulario o realizar acciones específicas.
                        5. <label>: Para asociar etiquetas descriptivas a los controles.
                        6. <fieldset>: Para agrupar controles relacionados.
                        7. <legend>: Para proporcionar una etiqueta descriptiva para un <fieldset>.
                        8. <input type="radio">: Para seleccionar una opción entre varias.
                        9. <input type="checkbox">: Para seleccionar una o varias opciones.
                        10. <input type="submit">: Para enviar el formulario.
                        11. <input type="reset">: Para restablecer los valores del formulario.

  14. Ciclo de Vida de los Sistemas de Información (CVSI):

    El CVSI describe las etapas desde la concepción hasta el desmantelamiento de un sistema, incluyendo investigación, análisis de requisitos, diseño, desarrollo, operación, mantenimiento y desmantelamiento.

  15. Ciclo de Desarrollo de Sistemas de Información (CDSI):

    El CDSI se centra específicamente en la construcción y entrega del sistema, con etapas como recolección de requisitos, diseño, implementación, pruebas, despliegue y mantenimiento.
